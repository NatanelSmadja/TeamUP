# TEAMUP V2.7 — מערכת עם מספר קבוצות

## לפני העדכון
בצע גיבוי ב-Git:

```bash
git add .
git commit -m "Backup before multi group update"
git push
```

## התקנה
1. עצור את השרת עם `Ctrl + C`.
2. העתק מהעדכון את התיקיות `src` ו-`supabase` אל הפרויקט ואשר החלפה.
3. ב-Supabase SQL Editor הרץ את:

```text
supabase/migrations/011_multi_group_platform.sql
```

4. הפעל:

```bash
npm run dev
```

אין צורך ב-`npm install`.

## מה משתנה
- כל המשתמשים והמידע הקיימים נשארים בקבוצה הקיימת, ששמה משתנה ל-`כוכבי סטאר בול`.
- משתמש חדש לא מצורף אוטומטית לשום קבוצה.
- משתמש יכול לבקש להצטרף לקבוצה קיימת או ליצור קבוצה חדשה.
- יוצר קבוצה מקבל תפקיד מנהל קבוצה.
- מנהל קבוצה מאשר או דוחה בקשות הצטרפות.
- מנהל קבוצה יכול לערוך את שם הקבוצה, התיאור והמיקום.
- משתמש יכול להיות חבר במספר קבוצות ולעבור ביניהן.
- משחקים, סקרים, דירוגים, התראות וחברי סגל נשארים נפרדים לכל קבוצה.
- מנהל קבוצה יכול להעניק הרשאות למנהלים נוספים בתוך הקבוצה שלו.
